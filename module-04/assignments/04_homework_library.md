## Часть 1. Заполнение данными (15 баллов)

### Задание:

Создайте SQL-скрипт для заполнения базы данных `library_final` (из Модуля 3) тестовыми данными.

### Требования к данным:

| Таблица | Количество | Особенности |
|---------|------------|-------------|
| authors | 30 | Разные страны, эпохи |
| publishers | 15 | Реальные издательства |
| categories | 20 | С иерархией (родительские/дочерние) |
| books | 100 | Разные годы, цены, статусы |
| book_authors | 120 | Связи M:N |
| book_categories | 150 | Связи M:N |
| book_copies | 200 | Разные статусы и состояния |
| readers | 50 | Реалистичные данные |

| fines | 30 | Разные статусы оплаты |

### Требования к скрипту:

- Использование множественной вставки (INSERT ... VALUES (), (), ())
- Использование INSERT ... SELECT где уместно
- Реалистичные данные (не test1, test2)
- Корректные связи между таблицами
- Разнообразные даты (прошлые, текущие)

## Часть 2. Запросы выборки (15 баллов)

### Задание:

Создайте SQL-скрипт с запросами для отчётности библиотеки.

### Обязательные запросы:

1. **Все книги с авторами и категориями**
   - Название, авторы (через запятую), категории, год, цена

2. **Доступные книги**
   - Книги, у которых есть свободные экземпляры

3. **Читатели с активными выдачами**
   - ФИО, название книги, дата выдачи, дата возврата

4. **Просроченные выдачи**
   - Читатель, книга, дата выдачи, дата возврата, дней просрочки

5. **Статистика по читателям**
   - ФИО, всего выдач, активных, просроченных, сумма штрафов

6. **Популярные книги**
   - Топ-20 книг по количеству выдач

7. **Новые поступления**
   - Книги, добавленные за последний месяц

8. **Должники**
   - Читатели с неоплаченными штрафами

9. **Статистика по категориям**
   - Категория, количество книг, количество выдач

10. **История выдачи книги**
    - Все выдачи для конкретной книги с читателями и датами

## Часть 3. Операции обновления (10 баллов)

### Задание:

Реализуйте следующие операции обновления:

1. **Автоматическое обновление статуса выдачи:**
   ```
   Если due_date < CURDATE() AND status = 'active'
   То status = 'overdue'
   ```

2. **Расчёт штрафов:**
   ```
   Для всех overdue выдач создать/обновить штраф
   10 рублей за каждый день просрочки
   ```

3. **Обновление рейтинга книг:**
   ```
   Установить rating = среднее значение отзывов
   ```

4. **Массовое изменение статуса:**
   ```
   Книги, не выдававшиеся 5 лет → status = 'archived'
   ```

5. **Обновление статистики читателей:**
   ```
   Пересчитать total_loans, active_loans для каждого читателя
   ```

## Часть 4. Предикаты и фильтрация (5 баллов)

### Задание:

Создайте запросы с использованием всех изученных предикатов:

1. **LIKE:** Поиск книг по шаблону
2. **BETWEEN:** Книги в диапазоне годов/цен
3. **IN:** Фильтрация по списку категорий/авторов
4. **IS NULL:** Записи с отсутствующими данными
5. **EXISTS/NOT EXISTS:** Проверка связанных записей
6. **REGEXP:** Поиск по регулярному выражению

---

## Часть 5. Отчётность (5 баллов)

### Задание:

Создайте сложные запросы для отчётности:

1. **Ежемесячный отчёт:**
   - Количество выдач по месяцам за год
   - Сумма штрафов по месяцам

2. **Отчёт по авторам:**
   - Количество книг, выдач, средний рейтинг

3. **Отчёт по категориям:**
   - Популярные категории за период

4. **Сводный отчёт библиотеки:**
   - Всего книг, читателей, активных выдач, должников
