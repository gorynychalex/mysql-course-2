# Домашнее задание - Модуль 5

## Тема: Сложные запросы для библиотеки

### Легенда

Для системы библиотеки необходимо реализовать сложные запросы для отчётности и аналитики.

---

## Часть 1. UNION и комбинирование данных (10 баллов)

### Задание:

Реализуйте запросы:

1. **Единый список людей:** Объедините читателей и сотрудников с указанием типа
2. **История активности:** Объедините выдачи и резервирования в одну временную шкалу
3. **Поиск по каталогу:** Поиск одновременно по книгам, авторам и категориям
4. **Статусы экземпляров:** Объедините информацию о доступных, выданных и зарезервированных книгах

---

## Часть 2. Подзапросы (15 баллов)

### Задание:

Создайте запросы с подзапросами:

1. Книги с ценой выше средней по категории
2. Читатели, бравшие книги определённого автора
3. Авторы, у которых все книги дороже 500 рублей
4. Категории с количеством книг больше среднего
5. Книги, которые брали все читатели из определённого города

---

## Часть 3. EXISTS для проверок (10 баллов)

### Задание:

Используйте EXISTS для:

1. Найти читателей с активными выдачами
2. Найти книги без свободных экземпляров
3. Найти авторов без книг в наличии
4. Найти категории без книг
5. Найти читателей без штрафов

---

## Часть 4. Представления (15 баллов)

### Задание:

Создайте представления для отчётности:

1. `v_full_book_info` - полная информация о книгах
2. `v_loan_history` - история выдач
3. `v_reader_activity` - активность читателей
4. `v_finance_report` - финансовый отчёт (штрафы)
5. `v_inventory_status` - состояние фонда
