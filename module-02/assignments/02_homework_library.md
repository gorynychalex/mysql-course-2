# Домашнее задание - Модуль 2

## Тема: Проектирование базы данных библиотеки

### Легенда

Продолжаем разработку системы управления библиотекой. На этом этапе необходимо спроектировать полноценную схему базы данных с учётом всех связей и нормализации.

## Часть 1. Анализ требований (5 баллов)

### Задание:

Изучите требования к системе и определите сущности:

#### Требования к системе библиотеки:

1. **Учёт книг:**
   - Книга имеет название, автора, ISBN, год издания
   - У книги может быть несколько авторов
   - Книга относится к одной или нескольким категориям
   - У книги есть статус (доступна, выдана, зарезервирована, в ремонте)
   - У книги есть инвентарный номер (уникальный для каждого экземпляра)

2. **Учёт читателей:**
   - Читатель имеет ФИО, дату рождения, контакты
   - У читателя есть номер билета (уникальный)
   - Читатель может иметь статус (активен, заблокирован)

3. **Выдача книг:**
   - Фиксируется дата выдачи и планируемая дата возврата
   - Фактическая дата возврата
   - Штраф за просрочку

4. **Резервирование:**
   - Читатель может зарезервировать книгу
   - У резерва есть срок действия

### Отчет:
Составьте список сущностей с атрибутами:

```
Сущность 1:
- атрибут 1 (тип, ограничения)
- атрибут 2 (тип, ограничения)
...

Сущность 2:
...
```

## Часть 2. Создание ER-диаграммы (15 баллов)

### Задание:

1. Откройте MySQL Workbench
2. Создайте новую модель `library_design.mwb`
3. Спроектируйте базу данных со следующими таблицами:

#### Обязательные таблицы:

| Таблица | Описание |
|---------|----------|
| `authors` | Авторы книг |
| `books` | Книги (библиографическая запись) |
| `book_copies` | Конкретные экземпляры книг |
| `categories` | Категории/жанры книг |
| `readers` | Читатели |
| `loans` | Выдача книг |
| `reservations` | Резервирование книг |
| `fines` | Штрафы |

#### Требования к связям:

- Автор ↔ Книги: **M:N** (один автор может написать много книг, у книги может быть несколько авторов)
- Книги ↔ Категории: **M:N** (книга может быть в нескольких категориях)
- Книги ↔ Экземпляры: **1:M** (одна библиографическая запись — много физических копий)
- Читатели ↔ Выдачи: **1:M** (один читатель — много выдач)
- Экземпляры ↔ Выдачи: **1:M** (один экземпляр — много выдач во времени)
- Читатели ↔ Резервы: **1:M**
- Экземпляры ↔ Резервы: **1:M**
- Читатели ↔ Штрафы: **1:M**

### Отчет:
- Файл `library_design.mwb`
- Экспорт диаграммы в PNG/PDF

## Часть 4. Реализация схемы (10 баллов)

### Задание:

1. **Экспортируйте SQL-скрипт** из Workbench (Forward Engineer)

2. **Требования к скрипту:**
   - Кодировка: utf8mb4_unicode_ci
   - Движок: InnoDB для всех таблиц
   - FOREIGN KEY с правильными ON DELETE/ON UPDATE
   - Индексы на внешних ключах
   - UNIQUE ключи где необходимо
   - DEFAULT значения где уместно
   - CHECK ограничения (если поддерживаются)
